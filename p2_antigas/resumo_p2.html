Notacao AT&T — Anatomia de um Comando

Sufixos, prefixos e posicoes src/dst — leia antes de qualquer outra coisa

Todo comando Assembly em AT&T segue o mesmo padrao:

instrucao[sufixo]   operando_fonte,   operando_destino
                         (src)               (dst)
AT&T e o inverso do Intel: o destino fica sempre a direita. addl %ecx, %eax significa %eax = %eax + %ecx, nao o contrario. O resultado vai para o operando da direita.

Sufixos: tamanho do dado operado

SufixoTamanhoTipo C correspondenteRegistrador ex.
b8 bits (1 byte)char%al, %dil, %r9b
w16 bits (2 bytes)short%ax, %si, %r9w
l32 bits (4 bytes)int%eax, %edi, %r9d
q64 bits (8 bytes)long / ponteiro%rax, %rdi, %r9
O sufixo da instrucao deve sempre combinar com o tamanho do tipo que voce esta operando em C. Se voce tem um int, usa movl / addl / cmpl. Se e short, usa movw / cmpw, e assim por diante.

Instrucoes com um unico operando (unarias)

Algumas instrucoes tem apenas um operando, que e ao mesmo tempo fonte e destino:

negl  %eax       # %eax = -%eax    (src e dst sao o mesmo)
incl  %ecx       # %ecx = %ecx + 1
decw  %si        # %si  = %si - 1
imulb y          # %al  = %al * y   (imulb: dst implicito e sempre %al)

O prefixo de um operando diz ao Assembly de onde vem o dado ou onde ele sera escrito:

Exemplos anotados

                   src           dst
movl    $5,         ↑    %ecx    ↑    # %ecx = 5            ($5 = constante)
movl    i,               %eax         # %eax = valor de i   (i sem $ = valor da global)
movq    $i,              %r8          # %r8  = endereco de i ($i com $ = endereco)

movl    (%r9),           %edi         # %edi = *r9           ((%r9) = desreferencia: le)

movq    -8(%rbp),        %rdi         # %rdi = pilha[-8]     (restaurar apos call)
movl    32(%rdi),        %eax         # %eax = *(rdi+32)     (offset fixo = arr[8] se int[])
Memoria para memoria e proibido. Nao existe instrucao que leia de uma variavel global e escreva diretamente em outra. Sempre use um registrador no meio: movl i, %eax + movl %eax, j.
Resumo rapido: $ antes de letra = endereco. Sem $ = valor. % = registrador. ( ) = desreferencia (vai ate o endereco e acessa a memoria).

Quando voce chama uma funcao, voce carrega os argumentos nesses registradores antes do call. Quando voce recebe uma chamada, os parametros ja chegam nesses registradores automaticamente.

Quando nao ha chamada de funcao rolando, esses registradores nao tem proposito fixo. Use como quiser para variaveis locais e calculos intermediarios:

Regra de ouro antes de um call

Depois de qualquer call, assuma que %rax, %rcx, %rdx, %rdi, %rsi, %r8, %r9, %r10, %r11 foram destruidos. Se voce tem variaveis locais nesses registradores que precisa usar apos o call, salve na pilha antes (movq %reg, -N(%rbp)) e restaure depois.

Resumo: minha funcao tem uma variavel local int i, qual registrador uso?

Nao existe regra que force a escolha. A unica restricao real e: nao use os proibidos, e respeite os registradores fixos (args e retorno) quando houver calls. Na pratica:

  • Se i e um contador de loop: convencao e %ecx
  • Se i tambem e o 4o argumento de um call que vai acontecer em breve: use outro registrador para nao conflitar
  • Se precisar de varios temporarios ao mesmo tempo: distribua entre %r10, %r11, %r8, %r9
  • Documente com comentario: # int i -> %ecx no inicio da funcao

Secao .data — Variaveis Globais

Como declarar variaveis e arrays globais em Assembly

Toda variavel global precisa de 3 itens: .globl nome, .align N e a diretiva com o valor.

Regra de alinhamento: .align N onde N e o tamanho do tipo. char=1, short=2, int=4, long=8. Arrays usam o mesmo alinhamento do elemento.
Tipo CDiretiva ASAlignExemplo
char.byte1c: .byte -20
short.short ou .word2s: .short 1024
int.int ou .long4i: .int -256
long.quad8l: .quad 128
char[] / string.string1msg: .string "texto\n"

Exemplo completo

# C: char c = -20;  int i = -256;  long arr[3] = {1,2,3};

    .data

    .globl  c
    .align  1
c:  .byte   -20

    .globl  i
    .align  4

    .globl  arr

MOV e Casts (Conversao de Tipo)

Como o compilador traduz cada tipo de conversao em C

Quando origem e destino tem o mesmo tamanho, usa-se o sufixo correspondente:

InstrucaoTamanhoExemplo
movb8 bits (byte)movb $93, c
movw16 bits (word)movw $75, s
movl32 bits (long)movl $52, j
movq64 bits (quad)movq $4096, l

Enderecos sao sempre 64 bits

Sempre que for salvar o endereco de uma variavel ou array em um registrador, use movq e um registrador de 64 bits. Endereco de memoria em AMD64 ocupa 8 bytes — independente do tipo da variavel apontada.
Correto
# ptr = &i  (i eh int, mas o endereco e 64 bits)
movq  $i, %r8       # movq + reg 64 bits

# ptr = &arr[0]  (array de short)
movq  $arr, %r9     # movq + reg 64 bits

# ponteiro como parametro de funcao
movq  $r, %rcx      # 4o param int* -> %rcx (64 bits)
Errado
# ERRADO: movl so copia 32 bits
movl  $i, %r8d      # trunca o endereco!

# ERRADO: registrador de 32 bits
movl  $arr, %edi    # perde metade do endereco!

A unica excecao e mover o valor de uma variavel (sem o $), que obedece o tamanho do tipo:

# Valor de int i  -> movl (4 bytes)
movl  i, %eax

# Endereco de int i  -> movq (8 bytes)
movq  $i, %r8

movs = move com extensao de sinal. Usado quando o tipo C com sinal (char, short, int, long) vai para um tipo maior. A instrucao copia o bit mais significativo (bit de sinal) da origem e replica ele em todos os bits extras do destino. Assim, um numero negativo continua negativo e um positivo continua positivo apos a extensao.

O sufixo indica a conversao: a primeira letra e o tamanho da origem, a segunda letra e o tamanho do destino. Ex: movsbl = move sign-extended de byte para long (int de 32 bits).

InstrucaoConversaoExemplo CExemplo ASM
movsbwchar → shorts = c;movsbw c, %ax
movsblchar → intj = c;movsbl c, %eax
movsbqchar → longl = c;movsbq c, %rax
movswlshort → inti = s;movswl s, %edx
movswqshort → longl = s;movswq s, %rax
movslqint → longl = i;movslq %ecx, %rcx
movslq %ecx, %rcx e muito usado para preparar um indice int como offset de 64 bits antes de calcular enderecos de array.

movz = move com extensao de zero. Usado quando o tipo C sem sinal (unsigned char, unsigned short, unsigned int) vai para um tipo maior. A instrucao preenche todos os bits extras do destino com 0, ignorando o bit de sinal (que nao existe em tipos unsigned). Como valores unsigned sao sempre positivos, zerar a parte alta preserva o valor numerico.

Mesma logica de sufixo do movs: primeira letra = tamanho origem, segunda letra = tamanho destino. Ex: movzbq = move zero-extended de byte para quad (long de 64 bits).

InstrucaoConversaoExemplo CExemplo ASM
movzbwunsigned char → unsigned shortus = uc;movzbw uc, %ax
movzblunsigned char → unsigned intui = uc;movzbl uc, %eax
movzbqunsigned char → unsigned longul = uc;movzbq uc, %rax
movzwlunsigned short → unsigned intui = us;movzwl us, %eax
movzwqunsigned short → unsigned longul = us;movzwq us, %rax
movzlq NAO existe! Para converter unsigned int em unsigned long, basta usar movl normal. Em x86-64, escrever em um registrador de 32 bits zera automaticamente a parte alta do registrador de 64 bits correspondente. Esse comportamento e exclusivo do tamanho 32 bits. 
# unsigned long ul = ui;  (NAO existe movzlq)
movl  ui, %eax        # escrever em %eax ja zera os 32 bits altos de %rax
movq  %rax, ul        # agora %rax tem o valor com a parte alta zerada
Regra rapida movs vs movz: tipo C com sinal → movs (copia bit de sinal). Tipo C sem sinal → movz (preenche com zero). A unica excecao e unsigned int → unsigned long, que usa movl em vez de um movzlq inexistente.

Truncamento e o caminho contrario da extensao: tipo maior atribuido a tipo menor. Nao existe instrucao especial. Basta gravar no destino usando o registrador menor (a parte baixa do registrador maior). Os bits altos sao simplesmente descartados.

# c = i + j;  (resultado int truncado para char)
movl  i, %ecx
addl  j, %ecx    # soma em 32 bits
movb  %cl, c     # salva apenas os 8 bits baixos (%cl)

# us = ul;  (long truncado para short)
movq  ul, %rax
movw  %ax, us    # salva apenas os 16 bits baixos (%ax)
Voce escolhe a parte menor do registrador (%cl, %ax, %eax, etc.) compativel com o tamanho do tipo destino. Nada de movs/movz aqui.
Em Assembly, parenteses em volta de um registrador significam "va ate o endereco guardado nesse registrador e acesse a memoria". Isso e exatamente o que o operador * faz em C.
Operacao em CO que fazTraducao Assembly

Exemplo completo

/* C:
   long *ptr;
   ptr = &l;       // ptr aponta para l
   *ptr = 128;     // escreve 128 em l via ponteiro
   j = j + *iptr; // le o valor apontado por iptr
*/

# ptr = &l  (salva endereco de l no registrador %r10)
movq  $l, %r10          # movq porque endereco e sempre 64 bits

# *ptr = 128  (escreve 128 no endereco guardado em %r10)
movq  $128, (%r10)      # (%r10) = desreferencia: vai ate o endereco e escreve

# j = j + *iptr  (le o valor no endereco guardado em %rdx)
movl  j, %eax
addl  (%rdx), %eax      # (%rdx) = desreferencia: le o int no endereco
movl  %eax, j

O sufixo do mov combina com o tipo APONTADO, nao com o ponteiro

# short *ptr  ->  o ponteiro em si ocupa 64 bits (movq $var, %reg)
# mas ao LER ou ESCREVER via *ptr, o tamanho e o do short (movw)

movq  $s, %r11          # salva endereco de s (short) em %r11  ->  movq (64 bits)
movw  $128, (%r11)      # *ptr = 128  ->  movw porque *ptr e short (16 bits)

# int *iptr  ->  ao ler *iptr, usa movl (int = 32 bits)
movq  $i, %rdx          # salva endereco (64 bits)
movl  (%rdx), %eax      # le *iptr  ->  movl porque *iptr e int (32 bits)
Resumo: salvar o endereco no registrador = sempre movq (64 bits). Ler ou escrever via ponteiro = tamanho do tipo apontado (movb/movw/movl/movq).

Aritmetica

add, sub, imul, neg, inc, dec e suas variantes por tamanho

InstrucaoOperacaoSufixosExemplo
adddst = dst + srcb/w/l/qaddl %eax, s
subdst = dst - srcb/w/l/qsubl $5, %ecx
imuldst = dst * src (com sinal)w/l/qimulq $4, %r9  |  imulw %ax, %si

imulb tem formato diferente dos outros. Aceita apenas um operando (reg ou mem) sem constante direta. O operando implicito e sempre %al e o resultado fica em %al. Sempre mova o valor desejado para %al antes de chamar imulb. 
# C: res = x * y;   (x, y, res sao char)
movb  x, %al         # mover UM dos fatores para %al (obrigatorio)
imulb y              # %al = %al * y  (y: reg ou mem, nao constante)
movb  %al, res       # resultado sempre em %al

# Se um fator for constante, mover para registrador antes:
movb  $5, %cl        # imulb $5 nao existe -- mova para reg primeiro
movb  x, %al
imulb %cl            # %al = %al * %cl
Sintaxe AT&T (ordem invertida): addl src, dst significa dst = dst + src. O destino fica sempre a direita.

Exemplo: expressao aritmetica

# C: i = (i * 2) - (j + 5);
#    instrucao  src,   dst      resultado
movl  i,    %edx        # %edx = valor de i   (global 'i' sem $ = valor)
imull $2,   %edx        # %edx = %edx * 2     ($2 = constante; dst recebe o produto)
movl  j,    %edi        # %edi = valor de j
addl  $5,   %edi        # %edi = %edi + 5
subl  %edi, %edx        # %edx = %edx - %edi  (src=%edi, dst=%edx)
movl  %edx, i           # i    = %edx         (grava resultado na global)

Divisao (Aula 06.3)

idiv / div — instrucoes especiais que exigem registradores especificos

Divisao e diferente de todas as outras operacoes: o dividendo precisa estar num par de registradores especifico e o divisor NAO pode ser um valor imediato.

Divisao com sinal (tipos int, long, char, short)

32 bits (int)
# C: q = a / b;   r = a % b;
movl  a,    %eax  # %eax = valor de a  (dividendo: OBRIGATORIO ir para %eax)
cdq               # estende %eax para %edx:%eax  (sem operandos)
idivl b           # divide %edx:%eax por b  (b: reg, mem ou constante em reg)
                  # quociente -> %eax  (automatico, sem dst explicito)
                  # resto     -> %edx  (automatico)
movl  %eax, q     # q = quociente
movl  %edx, r     # r = resto
64 bits (long)
# C: q = a / b;   r = a % b;
movq  a,    %rax  # %rax = valor de a  (dividendo: OBRIGATORIO ir para %rax)
cqo               # estende %rax para %rdx:%rax  (sem operandos)
idivq b           # divide %rdx:%rax por b
                  # quociente -> %rax  (automatico)
                  # resto     -> %rdx  (automatico)
movq  %rax, q     # q = quociente
movq  %rdx, r     # r = resto

Divisao sem sinal (tipos unsigned)

# unsigned int: zeramos %edx antes em vez de cdq
movl   a, %eax
xorl   %edx, %edx   # zera %edx (sem extensao de sinal)
divl   b             # quociente -> %eax, resto -> %edx

Mecanismos de Controle

if/else, while, for, break — logica invertida nos jumps

Em Assembly, o if pula para o fim do bloco quando a condicao e FALSA. Entao o jump usado e sempre o OPOSTO da condicao C.

if (a <= b)cmpl b, %eaxjg end_ifpula se NAO (a <= b)

Atencao com cmpb/cmpw/cmpl/cmpq: use sempre o sufixo correspondente ao tamanho do dado sendo comparado. cmpb para char, cmpw para short, cmpl para int, cmpq para long.

Tabela completa de jumps

Como montar o cmpX: qual operando vai onde?

cmpl src, dst computa internamente dst − src para setar as flags. Isso significa:

  • O operando que representa o lado esquerdo da comparacao C vai no registrador (dst, posicao direita)
  • O operando que representa o lado direito da comparacao C vai como src (posicao esquerda)
C
if (stamp > values[i])
if (sum < timer)
if (i >= 0)
Assembly
cmpl  %r8d, %edi     # edi=stamp, r8d=values[i]  -> edi-r8d
cmpl  %r9d, %edi     # edi=sum,   r9d=timer      -> edi-r9d
cmpl  $0,   %ecx     # ecx=i,     $0             -> ecx-0
Regra pratica: carregue a variavel que esta a esquerda do operador em C dentro de um registrador. Ela vai na direita do cmpX. A variavel ou constante da direita do operador vai como src (esquerda do cmpX). Memoria-para-memoria e proibido: se ambos sao globais, carregue um em registrador primeiro.

Acessando Arrays

Formula de offset, aritmetica de ponteiros, arrays de tipos diferentes

O processador nao conhece arrays. Todo acesso a arr[i] e calculado manualmente: endereco = &arr + (i * sizeof(tipo)). 
# Acesso generico a arr[i]:
#  instrucao  src,      dst
movq   $arr,  %r8        # %r8 = ENDERECO de arr  ($arr com $ = endereco da global)

addq   %r8,   %r9        # %r9 = %r8 + %r9 = &arr[i]  (dois registradores: src=%r8, dst=%r9)
movX   (%r9), %reg       # %reg = *r9              ((%r9) = vai ao endereco em %r9 e le)

Quando o indice de um array nao e uma variavel simples, o calculo do endereco tem mais etapas. Dois padroes aparecem nas provas e labs:

Padrao 1: indice eh uma soma de variaveis — arr[i + j]

Primeiro calcule i + j, depois converta para 64 bits, depois multiplique pelo sizeof. Cuide os tipos: se i e int e j e short, converta j para int antes de somar.

# C: v4[i + j] += expr;   (i=int, j=short, v4=long[])

# Passo 1: calcular i + j como int
movswl  j, %r10d         # r10d = (int)j  (promove short para int)
addl    i, %r10d         # r10d = i + j

# Passo 2: calcular &v4[i+j]
movq    $v4, %r8
movslq  %r10d, %r9       # r9 = (long)(i + j)  -- indice SEMPRE em 64 bits
imulq   $8, %r9          # 8 bytes por long
addq    %r8, %r9         # r9 = &v4[i+j]

# Ler, operar e escrever via (%r9)
movq    (%r9), %rdi      # rdi = v4[i+j]  (valor atual)
addq    %rax, %rdi       # rdi += expr
movq    %rdi, (%r9)      # v4[i+j] = rdi

Padrao 2: indice eh o valor de outro array — arr1[arr2[i]]

Dois acessos encadeados: primeiro leia arr2[i] da memoria, depois use o valor lido como indice para arr1.

# C: v1[v2[index]] = (int)value;   (v1=int*, v2=long*, index=long)
# v1 em %rdi, v2 em %rsi, index em %rcx, value em %dl

# Passo 1: calcular &v2[index] e ler o valor
movq    %rcx, %r8        # r8 = index
imulq   $8, %r8          # 8 bytes por long (tipo de v2)
addq    %rsi, %r8        # r8 = &v2[index]
movq    (%r8), %r9       # r9 = v2[index]  -- este sera o proximo indice

# Passo 2: usar v2[index] como indice de v1
imulq   $4, %r9          # 4 bytes por int (tipo de v1)
addq    %rdi, %r9        # r9 = &v1[v2[index]]

# Escrever no endereco calculado
movsbl  %dl, %r8d        # r8d = (int)value  (cast char->int)
movl    %r8d, (%r9)      # v1[v2[index]] = r8d
Em ambos os casos: o indice final precisa estar em 64 bits (movslq) antes do imulq. Nao pule essa conversao, mesmo que o valor ja seja positivo.

Chamada de Funcao (System V ABI)

Pilha, call/ret, passagem de parametros, retorno, printf, passagem por referencia, salvamento de registradores

A pilha e uma regiao de memoria usada para guardar dados temporarios durante a execucao de funcoes. Ela cresce para baixo: enderecos maiores sao o "fundo" e enderecos menores sao o "topo".

RegistradorNomeFuncao
%rspStack PointerAponta sempre para o topo atual da pilha (o ultimo valor empilhado). Atualizado automaticamente por push/pop e manualmente por subq/addq
%rbpBase PointerAponta para a base do frame atual. Fixo durante a execucao da funcao. Serve de referencia para acessar variaveis locais e parametros via offset negativo (-8(%rbp), -16(%rbp)...)

pushq e popq

InstrucaoO que fazEquivalente manual
pushq %regEmpilha: decrementa %rsp em 8 e escreve o valor no novo toposubq $8, %rsp + movq %reg, (%rsp)
popq %regDesempilha: le o topo para o registrador e incrementa %rsp em 8movq (%rsp), %reg + addq $8, %rsp

Prologo e epilogo: montando o frame da funcao

Toda funcao comeca com um prologo que cria seu frame na pilha e termina com um epilogo que desfaz tudo:

Prologo (inicio da funcao)
pushq  %rbp          # salva %rbp do caller na pilha
movq   %rsp, %rbp    # %rbp agora aponta para o topo atual
subq   $N,   %rsp    # reserva N bytes para variaveis locais
                     # (so necessario se houver variaveis locais ou calls)
Epilogo (fim da funcao)
leave                # equivale a: movq %rbp, %rsp + popq %rbp
                     # desfaz tudo que o prologo fez
ret                  # volta para o caller (ver proximo topico)

Layout da pilha durante uma funcao

# Enderecos crescem para CIMA, pilha cresce para BAIXO
#
#  Endereco alto  ──────────────────────
#                  ...caller frame...  
#                 ──────────────────────  <- %rbp do caller (salvo pelo pushq)
#  -8(%rbp)        variavel local 1      
# -16(%rbp)        variavel local 2      
# -24(%rbp)        variavel local 3      
# -32(%rbp)        variavel local 4      <- %rsp aponta aqui (apos subq $32)
#  Endereco baixo  ──────────────────────
O subq $N, %rsp do prologo define quantos bytes de espaco existem na pilha para salvar variaveis. N deve ser multiplo de 16 para alinhamento correto (na pratica: conte quantos registradores vai salvar, multiplique por 8, arredonde para multiplo de 16).

Quando o processador executa um call ou ret, ele manipula a pilha automaticamente:

InstrucaoO que o processador faz automaticamente
call funcao1. Empilha o endereco de retorno (endereco da instrucao seguinte ao call) com pushq
2. Salta para o inicio de funcao
ret1. Desempilha o endereco de retorno com popq
2. Salta de volta para esse endereco (linha seguinte ao call no caller)

Sequencia completa de uma chamada

# === NO CALLER (quem chama) ===
movl   i,    %edi       # 1. carrega argumentos nos registradores
movw   v,    %si
call   process          # 2. push endereco de retorno + jmp process
movl   %eax, resultado  # 4. execucao volta aqui; retorno esta em %eax

# === NA FUNCAO CHAMADA (process) ===
process:
    pushq  %rbp          # 3a. prologo: monta o frame
    movq   %rsp, %rbp
    # ... corpo da funcao ...
    movl   %ecx, %eax    # 3b. coloca retorno em %eax
    leave                # 3c. epilogo: desfaz o frame
    ret                  # 3d. pop endereco de retorno + jmp de volta
Apos o ret, o %rsp volta exatamente ao valor que tinha antes do call. Por isso e fundamental que o prologo e epilogo sejam simetricos: tudo que foi empilhado deve ser desempilhado antes do ret.
Os primeiros 6 argumentos vao em registradores. O tamanho do registrador deve corresponder ao tipo do parametro no C, nao ao tipo do caller.

Como alterar o valor dos registradores ao chamar outra funcao

Antes do call, voce precisa recarregar os registradores de argumento com os valores que a funcao chamada espera, mesmo que eles ja contenham os parametros da funcao atual. Cada call sobrescreve %rdi, %rsi, %rdx, %rcx, %r8, %r9 com os argumentos novos.

# Funcao atual: int soma(int a, int b)
# a chegou em %edi, b chegou em %esi
# Precisa chamar: printf("%d\n", a + b)

addl   %esi, %edi        # %edi = a + b   (sobrescrevi o 1o param)
movl   %edi, %esi        # move o resultado para a posicao do 2o arg do printf
movq   $fmt, %rdi        # 1o arg do printf: string de formato (sobrescreve a)
movl   $0,   %eax
call   printf
Por que precisa fazer isso: a CPU nao sabe quem e parametro de quem. Os registradores %rdi, %rsi, %rdx, %rcx, %r8, %r9 sao o "canal" pelo qual qualquer funcao recebe seus argumentos. Quando voce chama outra funcao, ela vai ler esses mesmos registradores. Se voce nao reorganizar os valores antes do call, a funcao chamada vai receber lixo (os valores da funcao atual no lugar dos que ela esperava).
E os valores da funcao atual que ainda preciso depois do call? Apos qualquer call, assuma que %rax, %rcx, %rdx, %rdi, %rsi, %r8, %r9, %r10, %r11 foram destruidos pela funcao chamada. Se algum desses guardava uma variavel local que voce vai usar apos o call, salve na pilha antes (movq %reg, -N(%rbp)) e restaure depois. Detalhes no proximo accordion. 
# C: process(i, v, &vetB[i]);
# void process(int x, short v, int *pos)

# Calcula endereco de vetB[i]
movq   $vetB, %r8
movslq i, %r9
imulq  $4, %r9
addq   %r8, %r9          # r9 = &vetB[i]

movl   i, %edi           # 1o param: int x
movw   v, %si            # 2o param: short v
movq   %r9, %rdx         # 3o param: int* (endereco = 64 bits)
call   process
Passar por referencia = passar o ENDERECO ($variavel ou endereco calculado de array). Sempre 64 bits porque e um ponteiro.

Ler e escrever via ponteiro recebido como parametro

movl   %esi, %eax        # eax = v1
addl   %edx, %eax        # eax = v1 + v2
movl   %eax, (%rcx)      # *s = resultado  (escreve no endereco)
Registradores de argumento (%rdi, %rsi, %rdx, %rcx, %r8, %r9) e %rax, %r10, %r11 podem ser destruidos por qualquer chamada de funcao. Se voce tem variaveis locais nesses registradores que precisam sobreviver apos o call, salve-as na pilha antes e restaure depois.

Variaveis globais nao precisam ser salvas: basta reler da memoria apos o call. O padrao se aplica a variaveis locais armazenadas em registradores.

Passo a passo

  1. No prologo da funcao (logo apos movq %rsp, %rbp): reserve espaco na pilha com subq $N, %rsp, onde N = (numero de registradores a salvar) × 8
  2. Antes de cada call: mova cada registrador local para sua posicao na pilha
  3. Configure os argumentos do call nos registradores corretos
  4. Execute o call
  5. Apos o call: restaure os registradores da pilha de volta
  6. Capture o retorno de %rax (se houver)
C
long process(int stamp, long limit, char base) {
    int i;
    for (i = 7; i >= 0; i--) {
        if (stamp > values[i])
            limit = (2*limit) + aux((short)i, values + i);
        else
            printf("%d, %ld\n", stamp, limit);
    }
    return limit;
}
Assembly (estrutura de save/restore)
process:
    pushq   %rbp
    movq    %rsp, %rbp
    # 4 regs x 8 = 32 bytes
    subq    $32, %rsp

    # variaveis: stamp=%edi, limit=%rsi, i=%ecx
    movl    $7, %ecx

begin_for:
    cmpl    $0, %ecx
    jl      end_for

    # ... calcular values[i], comparar ...

    # === ANTES DE CHAMAR aux() ===
    movq    %rdi, -8(%rbp)    # salva stamp
    movq    %rsi, -16(%rbp)   # salva limit
    movq    %rcx, -24(%rbp)   # salva i
    movq    %r11, -32(%rbp)   # salva r11

    # carrega params de aux e chama
    movw    %cx, %di
    movq    $values, %rsi
    ... (calculo do endereco) ...
    call    aux

    # === APOS O CALL ===
    movq    -8(%rbp), %rdi    # restaura stamp
    movq    -16(%rbp), %rsi   # restaura limit
    movq    -24(%rbp), %rcx   # restaura i
    movq    -32(%rbp), %r11   # restaura r11

    # retorno de aux esta em %rax
    addq    %rax, %r11        # r11 = (2*limit) + aux(...)
    movq    %r11, %rsi        # limit = r11
Posicoes na pilha: o primeiro registrador salvo vai em -8(%rbp), o segundo em -16(%rbp), e assim por diante (sempre multiplos de 8). O subq $N, %rsp no prologo precisa ser grande o suficiente para cobrir todas as posicoes usadas — N = maior offset usado (ex: se usar ate -32(%rbp), N deve ser pelo menos 32).

Regras que nunca mudam

  • Mem → Mem e proibido: sempre use um registrador intermediario. movl j, %eax + movl %eax, i
  • Indice de array sempre em 64 bits: movslq %ecx, %r9 antes do imulq
  • Cast antes de operar: se os tipos sao diferentes, converta primeiro, depois opere
  • Sufixo da instrucao = tamanho do dado: b=byte(char), w=word(short), l=long(int), q=quad(long)
  • AT&T e invertido: addl src, dst significa dst = dst + src

Explicacao de cada ponto

Prologo: pushq %rbp / movq %rsp, %rbp / subq $32, %rsp

Prologo padrao da funcao. O subq $32, %rsp reserva 32 bytes na pilha para salvar 4 registradores callee-clobbered (8 bytes cada) antes dos calls a aux e printf. O espaco e alocado uma unica vez aqui no inicio para ser reutilizado nos dois calls.

movl $7, %ecx -- inicializacao do for

Inicializa i = 7. Como i e int (32 bits), usa movl e o registrador %ecx (parte 32-bit de %rcx). A escolha de %ecx segue a convencao de contador de loop.

cmpl $0, %ecx / jl end_for -- condicao do for

Avalia i >= 0. O cmpl calcula %ecx - 0 e seta flags. O jl end_for usa logica invertida: pula para sair do loop quando NAO vale i >= 0, ou seja, quando i < 0.

Bloco values[i] -- calculo de endereco

Formula geral: &values + i * sizeof(int) = &values + i * 4. Os 5 passos:

  1. movq $values, %r8 -- carrega o endereco base do array. Usa movq porque endereco e 64 bits, e $values com cifrao porque queremos o endereco, nao o valor.
  2. movslq %ecx, %r9 -- promove i de int (32b) para long (64b) com sign-extend. Indices de array sempre precisam ser 64 bits para usar em imulq.
  3. imulq $4, %r9 -- multiplica por sizeof(int) = 4.
  4. addq %r8, %r9 -- soma base + offset = endereco final de values[i].
  5. movl (%r9), %r8d -- desreferencia: vai ao endereco em %r9 e le um int (32 bits) para %r8d.

cmpl %r8d, %edi / jle else_if -- if (stamp > values[i])

Compara stamp - values[i]. Ambos sao int, entao cmpl. Logica invertida no salto: jle else_if pula para o else quando stamp <= values[i], ou seja, quando o if FALHA. Cai no ramo if quando stamp > values[i].

(2*limit) calculado ANTES do call aux

O professor faz movq %rsi, %r11 seguido de imulq $2, %r11 antes de salvar nada na pilha. Isso e proposital: o call aux vai destruir %rsi (caller-saved), entao calculamos 2*limit primeiro e guardamos em %r11, que tambem vai ser salvo na pilha.

Salva 4 registradores na pilha antes do call

Como aux pode destruir qualquer registrador caller-saved (incluindo %rdi, %rsi, %rcx e %r11), salvamos os 4 que carregam nossas variaveis vivas: stamp, limit, i e (2*limit) acumulado.

movw %cx, di -- (short)i

O cast (short)i e um truncamento puro: pega os 16 bits baixos de %ecx (que sao %cx) e move para %di (parte 16-bit de %rdi). Nao precisa de sign-extend nem zero-extend, porque o destino e exatamente 16 bits. Nota: o gabarito tem typo e escreve di sem o %; o assembler nao compila assim.

Calculo de (values + i) como segundo argumento

O segundo parametro de aux e int *slice (ponteiro). O proprio C ja resolve a aritmetica de ponteiro: values + i equivale a &values[i]. Em assembly: base + i * sizeof(int) = base + i*4, usando movslq + imulq $4 + addq, com resultado em %rsi (porque ponteiros = 64 bits).

call aux + restauracao

Apos o call, restauramos os 4 registradores salvos. O retorno de aux e long, fica em %rax. Em seguida addq %rax, %r11 soma o retorno aos 2*limit ja calculados, e movq %r11, %rsi atualiza limit.

jmp end_if -- obrigatorio

Sem esse jmp, depois de executar o ramo if a execucao cairia direto no else_if: e executaria os dois ramos. O jmp end_if garante que apenas um dos ramos roda.

Ramo else: printf(stamp, limit) -- ordem dos argumentos

Esta e a parte mais sutil. O printf precisa de: %rdi = fmt, %esi = stamp, %rdx = limit. Mas no estado atual: %edi = stamp e %rsi = limit. A ordem importa:

  1. movq %rsi, %rdx -- limit (%rsi) para %rdx PRIMEIRO. Se fizessemos depois, %rsi ja teria sido sobrescrito.
  2. movl %edi, %esi -- stamp (%edi) para %esi. Pode fazer depois sem problema porque %esi e o que vamos sobrescrever, e ja salvamos limit em %rdx.
  3. movq $fmt, %rdi -- por ultimo o ponteiro do format string.

O movl $0, %eax e obrigatorio para chamar printf: a ABI exige que %al contenha o numero de argumentos float/double passados em registradores XMM. Como nao temos floats, vai zero.

decl %ecx -- i--

decl e a versao 32-bit de decrement. Equivale a subl $1, %ecx. Usa decl e nao decq porque i e int.

Epilogo: leave / ret

leave e atalho para movq %rbp, %rsp; popq %rbp -- desfaz o frame. ret faz pop do endereco de retorno e pula para la. O retorno (long) ja foi colocado em %rax com movq %rsi, %rax antes do leave.

Explicacao de cada ponto — modify

Prologo e reserva de 32 bytes

Mesma logica de Q1: 32 bytes na pilha para guardar 4 registradores callee-clobbered (%rdi, %rsi, %rdx, %rcx) antes de cada call. Reserva uma vez so e usa nos dois calls (calc e complex).

Inversao de argumentos para calc(v2, v1)

Esta e a parte mais sutil de modify. A funcao calc tem assinatura char calc(long *a, int *b), ou seja, espera v2 primeiro e v1 depois. Mas a funcao modify recebe na ordem (v1=%rdi, v2=%rsi). Precisamos trocar:

  1. movq %rdi, %r8 -- salva v1 em %r8 temporariamente (sem isso, perderiamos v1 ao sobrescrever %rdi).
  2. movq %rsi, %rdi -- v2 vai para o lugar do 1o argumento.
  3. movq %r8, %rsi -- recupera v1 e poe no lugar do 2o argumento.

cmpl %al, %dl -- avaliacao do while (BUG)

A intencao e comparar o retorno de calc (em %al, char) com value (em %dl, char). Ambos sao 8 bits. O correto seria cmpb %al, %dl. O professor escreveu cmpl, que nao compila com operandos de 8 bits. O jle end_while01 usa logica invertida: pula se NAO (value > calc), ou seja, sai do while quando value <= calc.

Ordem dos argumentos de complex(max, lim, v1)

Assinatura: long complex(int i, short j, int *v). Os args precisam ir para %edi (max), %si (lim) e %rdx (v1). Antes do call, v1 estava em %rdi. Para nao perder v1:

  1. movq %rdi, %rdx -- v1 vai PRIMEIRO para %rdx (3o arg), antes de %rdi ser sobrescrito.
  2. movl max, %edi -- carrega max (global) direto para %edi. max sem cifrao = valor da global, com cifrao seria o endereco.
  3. movw lim, %si -- carrega lim (global short de 16 bits) para %si.

v1[v2[index]] = (int)value -- acesso encadeado

Dois acessos a array, do mais interno para o mais externo:

  1. Calcular &v2[index]: v2 e long* (sizeof = 8). imulq $8, %r8 + addq %rsi, %r8 = endereco. Depois movq (%r8), %r9 le o long armazenado la.
  2. Usar v2[index] como indice de v1: v1 e int* (sizeof = 4). imulq $4, %r9 + addq %rdi, %r9 = endereco de v1[v2[index]].
  3. (int)value: value e char. Cast para int = sign-extend: movsbl %dl, %r8d.
  4. Escrita final: movl %r8d, (%r9) grava o int de 4 bytes no endereco calculado.

Explicacao de cada ponto — basic

movq $vet1, %rcx -- ptr = vet1

Inicializa o ponteiro local ptr. Como vet1 e um array global, seu nome em C ja decai para endereco; em assembly usamos $vet1 (com cifrao) para pegar o endereco do simbolo. movq porque ponteiros sao 64 bits.

Condicao while (sum < timer) -- promocao de char para int

timer e char (8 bits, signed). sum e int (32 bits). Nao se pode comparar diretamente: precisa promover timer para int. movsbl timer, %r9d le o byte e sign-extend para int. Depois cmpl %r9d, %edi + jge end_while02 -- logica invertida.

if (sum == 13) break -- jump direto para fim do while

O break e implementado como salto incondicional para o label de fim do while externo. Como queremos sair apenas quando sum == 13, usa-se je end_while02 (jump if equal).

Loop for interno (i de 0 a 9, sum++ junto)

Note o i++, sum++ no pos-passo: as duas variaveis sao incrementadas a cada iteracao (incl %r8d e incl %edi). O i nao podia ficar em %ecx porque ele ja armazena ptr -- por isso vai para %r8d.

Ordem dos args de printf -- BUG do gabarito

O printf espera %rdi = fmt, %esi = i, %edx = sum. O professor escreveu:

  • movq %ecx, %esi -- DOIS bugs: (1) movq com 32-bit nao compila; (2) %ecx tem ptr, nao i. O correto seria movl %r8d, %esi.
  • movq %edi, %edx -- mesmo erro de tamanho. O correto seria movl %edi, %edx.

A logica de ordem esta certa em principio: sum (do %edi) para %edx ANTES de sobrescrever %rdi com $fmt. So que o registrador do i esta errado.

modify(vet2, ptr, timer) -- preparacao dos args

Assinatura: void modify(int *v1, long *v2, char value):

  1. movq $vet2, %rdi -- endereco de vet2 para %rdi (1o arg).
  2. movq %rcx, %rsi -- ptr para %rsi (2o arg). ptr ja era ponteiro, basta copiar.
  3. movb timer, %cl -- carrega o byte de timer para %cl (3o arg, char). Para um char, %cl e a parte 8 bits de %rcx. Note que isso destroi o byte baixo do ptr que estava em %rcx -- por isso ptr ja foi copiado para %rsi antes (e tambem salvo em -16(%rbp) no inicio).

ptr++ -- aritmetica de ponteiro long*

addq $8, %rcx. Como ptr e long*, incrementar 1 posicao = avancar sizeof(long) = 8 bytes.

Loop ausente (BUG)

Apos o ptr++, o codigo cai direto em end_while02: sem nenhum jmp begin_while02. Isso significa que o while externo executa uma unica iteracao e termina. Logicamente esta errado -- o codigo C exige iterar enquanto sum < timer.

  • SAR vs SHR: bias e char (tipo com sinal), entao >> vira sarl (shift aritmetico), nao shrl. Usar shrl aqui seria errado e perderia pontos.

  • bias como indice: char nao pode ser usado direto em imulq. Precisa de dois passos: movsbl (char->int) depois movslq (int->long).
  • jmp antes do else: obrigatorio o jmp end_if1 no final do ramo if para nao cair no else.

    movq    %r8, -16(%rbp)          # salva endereco na pilha (call vai destruir %r8)

    movl    swap,   %esi            # 2o param: swap (int) -> %esi
    movq    factor, %rdi            # 1o param: factor (long) -> %rdi
    call    upper                   # retorno int em %eax

    movq    -8(%rbp), %rcx          # restaura ptr
    movq    -16(%rbp), %r9          # recupera &vetor[factor]
    movw    %ax, (%r9)              # vetor[factor] = (short)upper  (%ax = 16b baixos = truncamento int->short)

    # --- if (swap > 0) ---
    movl    swap, %r10d             # %r10d = swap
    cmpl    $0, %r10d               # computa swap - 0
    jle     end_if                  # pula se NAO (swap > 0), ou seja, se swap <= 0

    # --- swap = i ^ 0xFFEEDDCC ---
    movl    i, %r10d                # %r10d = i  (le global)
    xorl    $0xFFEEDDCC, %r10d     # %r10d = i XOR 0xFFEEDDCC  (xorl = XOR 32 bits)
    movl    %r10d, swap             # swap = resultado

end_if:
    # pos-passo do for: i++
    movl    i, %r10d                # le global i
    incl    %r10d                   # i++
    movl    %r10d, i                # escreve de volta (i e global, nao registrador)

    jmp     begin_for

end_for:
    leave
    ret
Pontos criticos desta questao:
  • ptr = &vetor[i] + 2: o +2 e aritmetica de ponteiro sobre short*, entao avanca 2 * sizeof(short) = 4 bytes, nao 2. Usar addq $4, %r9.
  • *ptr como argumento: ptr esta em %rcx, mas %rcx e necessario para o 4o argumento de mixer. Salve ptr na pilha antes, use %r10 para recuperar e desreferenciar.
  • Ordem dos args de mixer: 4 argumentos. O 4o (long factor) vai para %rcx. Prepare %rcx por ultimo, pois %rcx tambem e onde ptr esta guardado.
  • (short)upper(...): upper retorna int (%eax). Truncamento para short = pegar os 16 bits baixos via movw %ax, (%r9).
  • i e global: nao esta em registrador fixo. A cada leitura: movl i, %r10d. A cada escrita: movl %r10d, i. O incremento i++ tambem passa pela memoria.
  • XOR com constante grande: xorl $0xFFEEDDCC, %r10d -- constante cabe em 32 bits (int), entao xorl funciona diretamente.
